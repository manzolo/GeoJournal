📍 Posizione GPS

L'app richiede il permesso di localizzazione in due casi, entrambi avviati esplicitamente dall'utente:

  • Rileva posizione — lettura singola della posizione corrente per aggiungere un punto geografico.
  • Tracciamento percorso — servizio in foreground (icona nella barra di stato e notifica visibile) che registra il percorso GPS finché l'utente non lo ferma. Al termine genera un file KML salvato localmente, mai trasmesso automaticamente.

La posizione non viene mai raccolta in background silenzioso né trasmessa automaticamente a server esterni.

  • Firebase Auth (Google LLC) — autenticazione opzionale
  • Firebase Firestore (Google LLC) — sincronizzazione metadati cloud opt-in (punti, promemoria, visite); Firebase Storage — foto opt-in
  • Firebase Crashlytics (Google LLC) — segnalazioni di crash anonime
  • OpenStreetMap / OSMDroid — tiles cartografiche, nessun dato personale trasmesso
  • Google Drive — usato solo se l'utente configura esplicitamente un URI di destinazione tramite la selezione file di Android. Una volta configurato, il backup avviene automaticamente ogni notte (tra le 02:00 e le 06:00), solo in presenza di connessione di rete e batteria non scarica. L'app non accede a nessun altro file Drive oltre alla destinazione scelta dall'utente.

Policy di Google: policies.google.com/privacy

🔍 What GeoJournal collects

✅ Guest mode (recommended for privacy): GeoJournal works fully offline without sign-up. Your data stays only on your device — no server, no cloud, not even I as the developer can see it. Export a periodic backup from Profile → Export backup to keep your data safe.

🔓 You don't have to take my word for it: GeoJournal is open source. You can read the code on GitHub and verify it yourself.

Only if you choose to sign in with Google, you can choose what to sync to Firebase — a Google service, not my own server. Nothing is sent automatically: every data type is off by default and can be enabled individually from Profile → Sync & Privacy. Other users can never access your data. However, as the developer I have technical admin access to the Firebase console: I do not view your data and have no interest in doing so, but technically I could. If maximum privacy matters to you, use guest mode.

  • Email and name — provided by Google, used only to identify the account
  • User identifier (UID) — generated by Firebase Auth, never shared with third parties
  • Geo points — title, description, GPS coordinates, tags, emoji, rating — saved to Firebase Firestore only when signed in and only if the toggle is enabled
  • Photos — stored on your device; uploaded to Firebase Storage only when signed in and only if the "Photos" toggle is enabled (OFF by default)
  • Reminders — title, dates, type — saved to Firebase Firestore only when signed in and only if the toggle is enabled
  • Visit log — date and note of each visit — saved to Firebase Firestore only when signed in and only if the toggle is enabled

Automatically (even in guest mode):

  • Crash reports — Firebase Crashlytics collects anonymous crash data (device model, OS version, stack trace). No personal data included.

📍 GPS Location

The app requests location permission in two cases, both triggered explicitly by the user:

  • Detect location — a one-time position reading used to add a new geo point.
  • Route tracking — a foreground service (with a persistent status-bar icon and visible notification) that records a GPS route until the user stops it. When stopped, it generates a KML file saved locally, never transmitted automatically.

Location is never collected silently in the background or automatically transmitted to external servers.

☁️ Third-party services

  • Firebase Auth (Google LLC) — optional authentication
  • Firebase Firestore (Google LLC) — opt-in cloud metadata sync (points, reminders, visits); Firebase Storage — opt-in photo backup
  • Firebase Crashlytics (Google LLC) — anonymous crash reporting
  • OpenStreetMap / OSMDroid — map tiles, no personal data transmitted
  • Google Drive — used only if the user explicitly configures a destination URI via Android's file picker. Once configured, backup runs automatically every night (between 02:00 and 06:00), only when a network connection is available and the battery is not low. The app does not access any Drive files other than the destination chosen by the user.

Google Privacy Policy: policies.google.com/privacy

💾 Where data is stored

  • Local — Room database on the device (guest or signed-in mode)
  • Cloud opt-in — Firebase Firestore for points, reminders and visits; Firebase Storage for photos; all signed-in only and toggle enabled (OFF by default); Google servers in EU/USA
  • Local — Photos stored on device; uploaded only if the "Photos" toggle is explicitly enabled
  • Backup — ZIP file in the app's private storage (up to 5 files retained). If the user has configured a Drive destination, a copy is written automatically every night to the same chosen file, without creating additional files or accessing any other Drive data.

Firebase data is linked to your Google account and retained until you delete it. You can delete your account and all data directly from the app: Profile → "Delete account and all data". This permanently deletes all Firestore documents linked to your account. Photos, being local only, are removed when the app is uninstalled.
