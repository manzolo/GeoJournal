🔍 Cosa raccoglie GeoJournal

✅ Modalità ospite (consigliata per la privacy): GeoJournal funziona completamente offline senza registrazione. I tuoi dati restano solo sul tuo dispositivo — nessun server, nessun cloud, nemmeno io sviluppatore posso vederli. Esporta un backup periodico da Profilo → Esporta backup per non perdere i dati.

🔓 Non devi fidarti solo della mia parola: GeoJournal è open source. Puoi leggere il codice su GitHub e verificare di persona.

Solo se scegli di accedere con Google, puoi scegliere cosa sincronizzare su Firebase — un servizio di Google, non un server mio. Nulla viene inviato automaticamente: ogni tipo di dato è disattivato per default e può essere abilitato singolarmente da Profilo → Sincronizzazione & Privacy. Gli altri utenti non possono mai accedere ai tuoi dati. Tuttavia, in quanto sviluppatore ho accesso tecnico alla console Firebase: non consulto i tuoi dati e non ho alcun interesse a farlo, ma tecnicamente potrei. Se la massima privacy è importante per te, usa la modalità ospite.

  • Email e nome — forniti da Google, usati solo per identificare l'account
  • Identificatore utente (UID) — generato da Firebase Auth, mai condiviso con terzi
  • Punti geografici — titolo, descrizione, coordinate GPS, tag, emoji, valutazione — salvati su Firebase Firestore solo se loggato e solo se l'utente attiva il relativo toggle
  • Foto — salvate solo sul dispositivo, non vengono mai caricate su server o cloud
  • Note vocali (audio) — salvate solo sul dispositivo, non vengono mai caricate su server o cloud
  • Promemoria — titolo, date, tipo — salvati su Firebase Firestore solo se loggato e solo se il toggle è attivo
  • Registro visite — data e nota di ogni visita — salvato su Firebase Firestore solo se loggato e solo se il toggle è attivo

Automaticamente (anche in modalità ospite):

  • Report di crash — Firebase Crashlytics raccoglie dati anonimi di crash (modello dispositivo, versione OS, stack trace). Non contengono dati personali.

📍 Posizione GPS

L'app richiede il permesso di localizzazione solo quando l'utente aggiunge un punto e preme "Rileva GPS". La posizione non viene tracciata in background né trasmessa automaticamente.

☁️ Servizi di terze parti

  • Firebase Auth (Google LLC) — autenticazione opzionale
  • Firebase Firestore (Google LLC) — sincronizzazione metadati cloud opt-in (punti, promemoria, visite)
  • Firebase Crashlytics (Google LLC) — segnalazioni di crash anonime
  • OpenStreetMap / OSMDroid — tiles cartografiche, nessun dato personale trasmesso
  • Google Drive — usato solo se l'utente configura esplicitamente il backup su Drive tramite la selezione file di Android

Policy di Google: policies.google.com/privacy

💾 Dove vengono conservati i dati

  • Locale — Database Room sul dispositivo (modalità ospite o loggata)
  • Cloud opt-in — Firebase Firestore per punti, promemoria e visite; solo se loggato e toggle attivo; server Google nell'UE/USA
  • Locale — Foto e audio salvati solo sul dispositivo, mai trasmessi a server esterni
  • Backup — File ZIP su storage locale o Google Drive (include le foto), solo se l'utente lo configura

I dati su Firebase sono associati al tuo account Google e vengono conservati finché non li elimini. Puoi eliminare l'account e tutti i dati in autonomia dall'app: Profilo → "Elimina account e tutti i dati". Questa operazione cancella tutti i documenti Firestore associati al tuo account. Le foto, essendo solo locali, vengono eliminate insieme all'app.

🔍 What GeoJournal collects

✅ Guest mode (recommended for privacy): GeoJournal works fully offline without sign-up. Your data stays only on your device — no server, no cloud, not even I as the developer can see it. Export a periodic backup from Profile → Export backup to keep your data safe.

🔓 You don't have to take my word for it: GeoJournal is open source. You can read the code on GitHub and verify it yourself.

Only if you choose to sign in with Google, you can choose what to sync to Firebase — a Google service, not my own server. Nothing is sent automatically: every data type is off by default and can be enabled individually from Profile → Sync & Privacy. Other users can never access your data. However, as the developer I have technical admin access to the Firebase console: I do not view your data and have no interest in doing so, but technically I could. If maximum privacy matters to you, use guest mode.

  • Email and name — provided by Google, used only to identify the account
  • User identifier (UID) — generated by Firebase Auth, never shared with third parties
  • Geo points — title, description, GPS coordinates, tags, emoji, rating — saved to Firebase Firestore only when signed in and only if the toggle is enabled
  • Photos — stored on your device only, never uploaded to any server or cloud
  • Voice notes (audio) — stored on your device only, never uploaded to any server or cloud
  • Reminders — title, dates, type — saved to Firebase Firestore only when signed in and only if the toggle is enabled
  • Visit log — date and note of each visit — saved to Firebase Firestore only when signed in and only if the toggle is enabled

Automatically (even in guest mode):

  • Crash reports — Firebase Crashlytics collects anonymous crash data (device model, OS version, stack trace). No personal data included.

📍 GPS Location

The app requests location permission only when the user adds a point and taps "Detect GPS". Location is never tracked in the background or transmitted automatically.

☁️ Third-party services

  • Firebase Auth (Google LLC) — optional authentication
  • Firebase Firestore (Google LLC) — opt-in cloud metadata sync (points, reminders, visits)
  • Firebase Crashlytics (Google LLC) — anonymous crash reporting
  • OpenStreetMap / OSMDroid — map tiles, no personal data transmitted
  • Google Drive — used only if the user explicitly configures Drive backup via Android's file picker

Google Privacy Policy: policies.google.com/privacy

💾 Where data is stored

  • Local — Room database on the device (guest or signed-in mode)
  • Cloud opt-in — Firebase Firestore for points, reminders and visits; signed-in only and toggle enabled; Google servers in EU/USA
  • Local — Photos and audio stored on device only, never transmitted to external servers
  • Backup — ZIP file on local storage or Google Drive (includes photos), only if configured by the user

Firebase data is linked to your Google account and retained until you delete it. You can delete your account and all data directly from the app: Profile → "Delete account and all data". This permanently deletes all Firestore documents linked to your account. Photos, being local only, are removed when the app is uninstalled.

🔒 Your rights (GDPR)

If you are a European Union user, you can exercise all your rights directly in the app, without needing to contact us:

  • Rectification and access — edit or view your data at any time from the app
  • Portability — export everything via Profile → Export backup (ZIP file with data and photos)
  • Erasure — permanently delete your account and all data via Profile → "Delete account and all data"
  • Objection / Restriction — disable sync for each individual data type from Profile → Sync & Privacy, or use guest mode to send nothing at all

For any other formal request, contact us via GitHub.
